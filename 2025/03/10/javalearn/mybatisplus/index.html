注解

@TableName(“表名”)映射到数据库表

@TableId(“主键”)映射到数据库主键

默认是Id_worker 按照雪花算法
AUTO是主键自增,需要到数据库里去设置一下。
INPUT是手动输入主键
NONE是没有主键
UUID是UUID
ID_WORKER_STR是字符串表示法

@TableField(“字段名”)映射到数据库字段

update

用mapper里的updateById即可

自动填充

  • 需要在实体类里添加字段,update_time和create_time,数据库里也要添加这两个字段
  • 然后在实体类里添加注解@TableField(fill = FieldFill.INSERT)和@TableField(fill = FieldFill.INSERT_UPDATE)

需要注意的地方：字段类型要是LocalDateTime(应该和生成器的一致否则会为空,数据库内是datetime类型这个会自动填充没有关系)
代码可以去官网复制

乐观锁

乐观锁：顾名思义十分乐观，它总是认为不会出现问题，无论干什么都不去上锁！如果出现了问题，再更新值测试。
悲观锁：顾名思义十分悲观，它总是认为会出现问题，无论干什么都会上锁，然后再去操作！

乐观锁实现方式：

取出记录时，获取当前 version
更新时，带上这个 version
执行更新时， set version = newVersion where version = oldVersion
如果 version 不对，就更新失败

乐观锁需要配置插件,这里和官网的不太一样，使用的时候要注意一下

乐观锁的实现方式就是在表中添加一个 version 字段，使用 @Version 注解标记这个字段。

看起来很简单，所以暂时不实现了，等有需要的时候再来实现。
