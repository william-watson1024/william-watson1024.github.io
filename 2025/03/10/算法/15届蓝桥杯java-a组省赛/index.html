试题A:拼正方形

【问题描述】

小蓝正在玩拼图游戏，他有 7385137888721 个 2 × 2 的方块和 10470245 个 1 × 1 的方块，他需要从中挑出一些来拼出一个正方形，比如用 3 个 2 × 2 和 4 个 1 × 1 的方块可以拼出一个 4 × 4 的正方形，用 9 个 2 × 2 的方块可以拼出一个 6 × 6 的正方形，请问小蓝能拼成的最大的正方形的边长为多少。

思路解答

本题可以发现 2 × 2 的方块是完全平方数，所以用 2 × 2 的方块先拼起来，然后就可以用 1 × 1 的循环扩展。

试题B:召唤数学精灵

【问题描述】

数学家们发现了两种用于召唤强大的数学精灵的仪式，这两种仪式分别被称为累加法仪式 A(n) 和累乘法仪式 B(n)。

累加法仪式 A(n) 是将从 1 到 n 的所有数字进行累加求和，即：A(n) = 1 + 2 + · · · + n。累乘法仪式 B(n) 则是将从 1 到 n 的所有数字进行累乘求积，即：B(n) = 1 × 2 × · · · × n。

据说，当某个数字 i 满足 A(i) − B(i) 能被 100 整除时，数学精灵就会被召唤出来。

现在，请你寻找在 1 到 2024041331404202 之间有多少个数字 i，能够成功召唤出强大的数学精灵。

思路解答

通过观察可以发现当 i 大于等于 10 的时候，B(i) 的值会被 100 整除，所以问题就简化为找到 A(i) 的值，然后判断是否能被 100 整除。sum(1,200) 是 20100，所以就可以以 200 为一个周期进行计算。

试题D:回文数组

【问题描述】

输入一个长为 n 的数组，输出使其变得回文的最少操作次数。

在 1 次操作中，可以二选一：

  1. 选择一个元素，对其增减 1
  2. 选择相邻的两个元素，对其增减 1

输入输出示例

输入共 2 行，第 1 行含一个正整数 n，第 2 行含 n 个正整数 x：

输出一个整数表示答案：

解释：

第 1 次操作后：[2, 3, 3, 4]
第 2 次操作后：[3, 3, 3, 4]
第 3 次操作后：[4, 3, 3, 4]，已经是回文

范围

1 ≤ n ≤ 1 × 10^5
-1 × 10^6 ≤ x ≤ 1 × 10^6

思路解答

可以使用双指针法，从数组的两端向中间移动，每次先用操作1将两端的数变成相等，然后看arr[i+1]和arr[j-1]是不是和他们同号，如果同号就将一部分操作改为操作2，否则就继续操作1。

试题E:吊坠

【问题描述】

输入 n 个长为 m 的字符串，作为图的结点，每两个结点间都有一条带权边，权值为两字符串的最长公共子串的长度。

注意，本题中的字符串都可以旋转，如 “abba” 可以旋转变换为 “aabb”，从而 “abba” 和 “acca” 之间的边权值为 2。

选出最少的边让所有结点连通，且成本最大，输出这个成本。

输入输出示例

输入共 n + 1 行，第 1 行含两个正整数 n、m，随后 n 行各含一个长为 m 的字符串：

输出一个整数表示答案：

解释：连接 <1, 2>，<2, 3>，<2, 4>，边权和为 4 + 2 + 2 = 8。

范围

1 ≤ n ≤ 200
1 ≤ m ≤ 50

思路解答

首先求出边权，每个字符串重复一次自身就可以得到全部的旋转字符串，不过要注意旋转字符串的长度不能超过原字符串的长度，然后用动态规划实现lcs算法，最后用krustral算法求最小生成树。

试题F:砍柴

【问题描述】

A、B 两人轮流砍柴，轮到某一方砍柴时，设原木剩余长度为 x：

  • 如果 x 为 0 或 1，本方无法继续砍柴，从而输掉比赛。
  • 如果 x ≥ 2，本方可选择一个长度 2 ≤ p ≤ x，砍掉长为 p 的一段。并且p必须是质数。

共 n 段原木（n 局游戏），每局 A 先手，双方都采用最佳策略。

对每局游戏，如果 A 能够取胜，输出 1，否则输出 0。

输入输出示例

输入共 n + 1 行，第 1 行含一个正整数 n，随后 n 行各含一个正整数 x：

输出 n 行，每行含一个整数 0 或 1：

解释：

  • 对 x = 1，A 已无法砍柴，A 失败。
  • 对 x = 2，A 砍掉 p = 2，轮到 B 时 x = 0，已无法砍柴，A 获胜。
  • 对 x = 6，A 砍掉 p = 5，轮到 B 时 x = 1，已无法砍柴，A 获胜。

范围

1 ≤ n ≤ 10^4
1 ≤ x ≤ 10^5

思路解答

先用质数筛选出所有的质数，然后用DFS判断每个数是不是必胜态。
